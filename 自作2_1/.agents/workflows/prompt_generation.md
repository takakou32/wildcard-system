# プロンプト作成・派生ワークフロー

ユーザーからの日本語による指示（例: 「仰向けでM字開脚、アヘ顔のプロンプトを作って」「バックで激しく突かれているプロンプト」等）から、Antigravity（AI）は以下のフローに従ってプロンプトを自律的に構築してください。

## ステップ1: 要求内容の分析とコンポーネント分解
ユーザーからの日本語の指示を受け取ったら、必要な要素を以下の3カテゴリに分解します。
1. カメラアングル・構図 (Camera Angle & Composition)
2. ポーズ・動作・体位 (Pose, Action & Position)
3. 表情・視線 (Expression & Gaze)

## ステップ2: 既存の『プロンプト作成用ライブラリ』からの検索とベースの決定
AIは必ず `milf用/prompt_library.md` を参照（`view_file`ツールなどを利用）します。
新ルールとして**「連番の直接指定」は不要**です。AIはユーザーの日本語指示（例：「正常位」「バック」「騎乗位」「フェラ」など）を基に、ライブラリ内の **「2. プロンプトベース（テンプレート）一覧」** を自然言語検索・照合し、最も適したテンプレート（`40_se100` 等）を特定してベースとします。
- **ベースが特定できた場合:**
  そのベースとなる連番の構成要素（タグ）を取得し、それにユーザーの追加指示（「もっとアヘ顔に」等）を組み合わせて改変します。
- **完全に一致するベースがない場合:**
  ライブラリ内の「1. カテゴリ別頻出タグ」から流用できる構成要素（例: 仰向け=`on back, lying` / 股間視点=`pov crotch` 等）をピックアップし、ゼロベースで組み立てます。

## ステップ3: Danbooruタグ検索スクリプトの実行（必要に応じて）
ライブラリに存在しない特殊なタグや、ユーザーが日本語で指定した新しいシチュエーション・属性・アイテムなどがある場合、既存のタグ検索スクリプトを利用して正式なタグを特定します。

**実行コマンド例:**
```bash
python search_tags.py "検索したいキーワード（英語または日本語）"
```
※ 例: `python search_tags.py "アヘ顔"` → `ahegao` や `rolled eyes` などのDanbooruタグを取得。
（注意: LLMのコンテキスト溢れを防ぐため、結果が多い場合はキーワードを絞り込んで再検索してください）

## ステップ4: プロンプトの組み立てと出力
ステップ2で収集した「ライブラリ準拠の構成要素」と、ステップ3で取得した「Danbooruタグ」を正しく組み合わせ、最終的なプロンプトを出力します。

**組み立てルール:**
- ユーザーの厳守事項に従い、以下の順序で出力すること:
  1. 連番（例: ベースにした元の番号_派生番号、ベースがない場合は新規の採番）
  2. カメラアングル・構図
  3. ポーズ・動作・体位（※ 人数や構図に関わる `1girl`, `1boy`, `boy standing before girl`, `faceless male` などの構造的なキャラクタータグや、`sweat`, `heavy breathing`, `after fellatio` などの状態・シチュエーションタグも積極的にここに含めること）
  4. 表情・視線
- ⚠️ **照明、場所、キャラクターの「身体的特徴（髪色・瞳の色・服装など）」に関するプロンプトは出力対象に含めないこと。**

## ステップ5: ユーザーへの提案・確認
組み立てたプロンプトをユーザーに提示し、問題がないか確認します。必要に応じて微調整を行ってください。
