# プロンプト作成・派生ワークフロー

新しくプロンプトを作成する、または既存のプロンプト（例: `40_se100`）の派生を作成するよう指示された場合、Antigravity（AI）は以下のフローに従ってプロンプトを構築してください。

## ステップ1: 要求内容の分析とコンポーネント分解
ユーザーからの指示（例：「仰向けでM字開脚、アヘ顔のプロンプトを作って」など）を受け取ったら、必要な要素を以下の3カテゴリに分解します。
1. カメラアングル・構図 (Camera Angle & Composition)
2. ポーズ・動作・体位 (Pose, Action & Position)
3. 表情・視線 (Expression & Gaze)

## ステップ2: 既存の『プロンプト作成用ライブラリ』の参照
AIは必ず `milf用/prompt_library.md` を参照（`view_file`ツールなどを利用）し、ユーザーの指示に該当する構図やポーズ、表情が既に存在しないか確認します。
- **既存プロンプトの派生指示がある場合:**
  指定された連番（例: `40_se100`）のベースタグをライブラリから取得し、それをベースに構築・改変します。
- **指示された構図に近いものが存在する場合:**
  ライブラリ内の「カテゴリ別頻出タグ」や既存の連番から流用できる構成要素（例: 仰向け=`on back, lying` / 股間視点=`pov crotch` 等）をピックアップします。

## ステップ3: Danbooruタグ検索スクリプトの実行（必要に応じて）
ライブラリに存在しない特殊なタグや、ユーザーが日本語で指定した新しいシチュエーション・属性・アイテムなどがある場合、既存のタグ検索スクリプトを利用して正式なタグを特定します。

**実行コマンド例:**
```bash
python search_tags.py "検索したいキーワード（英語または日本語）"
```
※ 例: `python search_tags.py "アヘ顔"` → `ahegao` や `rolled eyes` などのDanbooruタグを取得。
（注意: LLMのコンテキスト溢れを防ぐため、結果が多い場合はキーワードを絞り込んで再検索してください）

## ステップ4: プロンプトの組み立てと出力
ステップ2で収集した「ライブラリ準拠の構成要素」と、ステップ3で取得した「Danbooruタグ」を正しく組み合わせ、最終的なプロンプトを出力します。

**組み立てルール:**
- ユーザーの厳守事項に従い、以下の順序で出力すること:
  1. 連番（例: 40_se100_衍生番号）
  2. カメラアングル・構図
  3. ポーズ・動作・体位
  4. 表情・視線
- ⚠️ **照明、場所、キャラクターに関するプロンプト（例: 1girl, milf, indoors, cinematic lightingなど）は出力対象に含めないこと。**

## ステップ5: ユーザーへの提案・確認
組み立てたプロンプトをユーザーに提示し、問題がないか確認します。必要に応じて微調整を行ってください。
