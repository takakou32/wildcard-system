# シチュエーションごとの設計書およびプロンプトの統合生成パイプライン

このワークフローは、ユーザーから新しいテーマやシチュエーション（例：「〇〇のシチュエーションでプロンプトを作って」）を与えられた際に、AIが自律的に詳細設計書を作成し、レビューを経て最終的なDanbooruタグプロンプトを出力する一連のプロセスです。

## 前提条件とルール（非常に重要！）
このワークフローで生成するプロンプトは、以下の「厳守事項」に必ず従います。
1. **出力対象のカテゴリと順序:**
   - 1.連番（**必須**: プロンプト文字列の先頭に `0001,` `0002,` ... と**4桁ゼロ埋め**の番号タグを付ける。generate forever で何番が選ばれたか追跡できるようにするため）
   - 2.カメラアングル・構図
   - 3.ポーズ・動作・体位・状態（※ `1girl`, `1boy`, `faceless male`, `sweat`, `heavy breathing` などのキャラクター構造と状態を示すタグは積極的にここに含める）
   - 4.表情・視線
2. **除外対象:**
   - 照明（cinematic lightingなど）、場所（indoorsなど）、およびキャラクターの「身体的特徴（髪色・服装・顔の造形など）」に関する指定は**出力しない**。
   - 白目(rolling eyes)やアヘ顔(ahegao, mind break)などの極端な表情指定は**絶対に出力しない**。
3. **特定構図のルール:**
   - 構図に男性が映る場合（顔や体が入る場合）は、必ず `faceless male` などの見えない指定を行うこと。

---

## ステップ1: テーマフォルダの作成と前提情報の読み込み
1. ユーザーから指定されたテーマ名（例: `milf用_ベッドルーム`）で、親ディレクトリ（例: `自作2_1/`）内に**新しいフォルダを作成**します。
2. ユーザーから提供された設定ファイル（`source.yaml`等）やテキストメモがある場合は、それを読み込み、どのようなフェーズ・流れが必要かを分析します。

## ステップ2: シーン設計書の作成とレビュー (`design_doc.md`)

### 2-0. 【必須】シーンごとのキーワード検索（新規作成禁止ルール）

設計書を書き始める前に、**各シーンについて必ず以下の手順を実行すること**。

1. そのシーンの核となるアクション・身体部位・状態を2〜4語のキーワードに分解する
   - 例:「挿入前にチンポをガン見する」→ `looking at penis`, `penis`, `wide eyes`, `挿入前`, `凝視`
2. そのキーワードで `verified_scene_patterns.md` の全文（セクションを問わず）を検索する
3. ヒットした候補をすべて列挙し、各エントリの **備考欄** を読んで文脈・強度・段階を比較する
4. 最も合致するIDを1件選定する

**【絶対ルール】** 上記検索を行った上でも合致するパターンが存在しない場合のみ、新規プロンプト作成を許可する。検索をスキップして新規作成することは禁止。

---

### 2-1. 設計書の作成

1. **`自作2_1/doc/verified_scene_patterns.md` を参照**し、2-0で選定したパターンを使用します。
2. 対象フォルダ内に `[テーマ名]_design_doc.md` を作成します。
3. 設計書の構成は以下のようにします:
   - 全体の共通設定（シチュエーション、フェーズ構成）
   - フェーズごとのシーン一覧
     - シーン名（例: 【1-1】○○の瞬間）
     - 使用するパターンID（`verified_scene_patterns.md` のエントリID）と、**その選定理由**（なぜ他の候補より適しているか）を日本語で記述。
     - パターンへの追加・変更が必要な場合はその差分を明記。
     - パターンが存在せず新規作成する場合は「既存なし（検索キーワード: ○○）」と明記すること。
4. **【重要】** 設計書が完成したら、一度ここで作業を止め、ユーザーに設計書のレビューを依頼します。（※この段階ではまだプロンプトを生成しません）
5. ユーザーから修正指示があれば設計書 (`design_doc.md`) を修正し、再度レビューを受けます。

## ステップ3: プロンプト生成 (`generated_prompts.yaml`) と保存
1. ユーザーから設計書 (`design_doc.md`) の内容にOK（承認）が出たら、次のステップに進みます。
2. 設計書で指定されたパターンIDのプロンプトを `verified_scene_patterns.md` から取得し、必要に応じて差分を適用してプロンプトを組み立てます。（日本語からタグへの変換は行わない）
3. 対象テーマフォルダ内に `generated_prompts_[テーマ名].yaml` を作成し、上述の「前提条件とルール」に完全に則ってプロンプトを出力します。
   - **【重要: YAMLのフォーマット】** 出力したプロンプトを「1ワードのワイルドカード」としてランダム出力できるよう、YAMLファイルは単一のルートキー（例: `disgust_to_pleasure:` や `theme_name:` など）を持ち、その下にリスト形式で各シーンのプロンプトを連ねる構成にしてください。
   - ※各シーンの前にコメント `# [連番] [日本語のシーン概要]` を忘れずに入れること。
4. 生成が完了したら、`notify_user` でユーザーに完了を報告し、作られたYAMLファイルの確認を促します。
   - **【重要: 完了時の出力】** このとき、ユーザーがすぐにコピペして使えるように、適切なフォルダパスを含めたワイルドカード文字列（例: `__wildcard-system/自作2_1/嫌悪から快感/disgust_to_pleasure__` や `__wildcard-system/自作2_1/[テーマフォルダ名]/[ルートキー]__` など）をメッセージ内に明記してください。
