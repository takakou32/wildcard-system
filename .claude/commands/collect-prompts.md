Easy Reforge で生成した PNG からプロンプトを収集し、`verified_scene_patterns.md` に自動分類・追加するフローを実行してください。
**ユーザーの操作は最後の承認のみ。** それ以外のステップはすべて自動で進めること。

## 引数

`$ARGUMENTS` にディレクトリパスが渡されます。空の場合はユーザーにパスを確認してください（これのみ例外）。

---

## 実行手順

### Step 1: セクション一覧の読み込み

`verified_scene_patterns.md` の冒頭 80 行を読み込み、「セクション一覧」テーブルを把握する。

---

### Step 2: プロンプト抽出

以下を実行する:

```
python 自作2_1/doc/scripts/extract_sd_prompts.py "<ディレクトリパス>"
```

- 結果が空（`[]`）の場合は「PNG ファイルまたはプロンプトが見つかりませんでした」と伝えて終了
- エラーの場合はエラー内容を表示して終了
- 処理中は「X 件の PNG を検出しました」と件数のみ表示する

---

### Step 3: シーンプロンプトの自動抽出（フィルタリング）

各エントリの positive prompt から以下のカテゴリを**除去**して「シーンプロンプト」を生成する。

**除去対象:**

| カテゴリ | 例 |
|---------|---|
| 画質・スタイル | `masterpiece`, `best quality`, `ultra-detailed`, `highres`, `8k`, `(extremely detailed CG` など |
| キャラクター外見 | 髪色・髪型、目の色、体型 (`large breasts`, `slim`, `mature female` など)、顔の特徴、キャラ名 |
| 服装・衣装 | `dress`, `shirt`, `skirt`, `kimono`, `bikini`, `yukata`, `school uniform` など（`nude`, `naked`, `topless`, `bottomless`, `panties` は残す） |
| ライティング | `soft lighting`, `cinematic lighting`, `rim light`, `dramatic lighting`, `backlighting`, `bloom` など |
| 場所・背景 | `bedroom`, `hotel room`, `living room`, `outdoor`, `beach`, `indoors`, `white sheets` など |
| LoRA・埋め込み | `<lora:...>`, `<embedding:...>`, モデル固有トークン |

**残す対象:** 人数・関係性・ポーズ・体位・アクション・アングル・表情・演出効果・身体フォーカス・状態タグ

シーンプロンプトが空または 5 タグ未満の場合はそのエントリをスキップ（追加候補から除外）する。

---

### Step 4: 自動分類・エントリ生成

各シーンプロンプトについて以下をすべて自動で決定する:

**a) カテゴリ判定**
Step 1 で読んだ「セクション一覧」を参照し、シーンプロンプトのキーワードから最も近いセクション・サブセクションを選ぶ。

**b) ID 採番**
対象サブセクションの「最終エントリID」に連番を振る（例: `35_bs004_9_1` → `35_bs004_9_2`）。
同一サブセクションへ複数追加する場合は連続して採番する。

**c) タイトル生成**
シーンプロンプトの主要タグ（体位・アクション・アングル・特徴的要素）から日本語タイトルを生成する。

**d) 説明文生成**
以下の形式で自動生成する:
`> アングル: {angle} / 動作: {action} / 表情: {expression} / 備考: {notes}`

**e) 静的/動的判定**
`{|}` または `__wildcard__` が含まれれば 🔀、なければ ✅。

**f) ソースタグ**
`[D]`（Direct）固定。

---

### Step 5: 追加サマリーの提示と承認

すべてのエントリが揃ったら、以下の形式でサマリーを表示してユーザーの承認を求める:

```
=== 追加予定エントリ ===

[1] ID: 35_bs004_10  ✅  [D]
    セクション: 1. 前戯・いちゃらぶ系 > 1-6. 添い寝・上から俯瞰
    タイトル: 添い寝・フレンチキス・手マン・蒸気
    プロンプト: on back, lying, (overhead shot, from above:1.5), ...
    > アングル: 真上俯瞰 / 動作: 添い寝・フレンチキス＋手マン / 表情: 照れ・蒸気 / 備考: ...

[2] ...

---
追加: X 件 / スキップ（シーン要素不足）: Y 件 / 対象総数: Z 件

承認しますか？ (y=全件追加 / n=キャンセル / 除外する番号をカンマ区切りで入力)
```

- `y` → Step 6 へ進む
- `n` → 終了
- 番号入力（例: `2,3`）→ 該当エントリを除外して再度サマリーを表示し、再確認する

---

### Step 6: verified_scene_patterns.md への一括追記

承認されたすべてのエントリを追記する。各エントリのフォーマット:

```markdown

**{ID}** — {タイトル} {✅/🔀} `{[source]}`
> アングル: {angle} / 動作: {action} / 表情: {expression} / 備考: {notes}
```
{scene_prompt}
```

---
```

挿入位置: 対象サブセクションの末尾（次の `### ` または `## ` の直前）。

追記完了後、「セクション一覧」テーブルの最終エントリIDを更新する。

---

### Step 7: 完了報告

```
=== 完了 ===
追加: X 件 / スキップ: Y 件
```
